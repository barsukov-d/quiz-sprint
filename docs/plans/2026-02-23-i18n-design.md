# i18n Design: English + Russian (vue-i18n@11 + Nuxt UI)

**Date:** 2026-02-23
**Branch:** `add-i18n`
**Scope:** Full app — all 27 Vue files

---

## Decision

**Library:** `vue-i18n@11` (Composition API mode, `legacy: false`)
**Nuxt UI integration:** `UApp :locale="locales[locale]"` — auto-switches Nuxt UI internal strings
**Languages:** `en` (default), `ru`

---

## File Structure

```
tma/src/i18n/
  index.ts          # createI18n instance + locale detection
  locales/
    en.ts           # English messages
    ru.ts           # Russian messages
```

---

## Locale Detection (priority order)

1. `localStorage.getItem('locale')` — manual override persisted across sessions
2. `retrieveLaunchParams().initData?.user?.languageCode` — Telegram user language
3. Fallback: `'en'`
4. Normalize: `languageCode` starts with `'ru'` → `'ru'`; anything else → `'en'`

## Language Switcher (ProfileView)

```vue
<USelect
  v-model="currentLocale"
  :items="[
    { label: 'English', value: 'en' },
    { label: 'Русский', value: 'ru' },
  ]"
  @update:model-value="setLocale"
/>
```

`setLocale(lang)`:
1. `locale.value = lang`
2. `localStorage.setItem('locale', lang)`

---

## Nuxt UI Internal Strings

`@nuxt/ui/locale` ships `en` and `ru` built-in. No custom locale definition needed.
`UApp :locale="locales[locale.value]"` auto-switches all Nuxt UI component strings.

---

## Interpolation Pattern

Dynamic values use named interpolation:

```typescript
// en.ts
quiz: { questionsCount: '{count} questions' }

// component
t('quiz.questionsCount', { count: quiz.questionsCount })
```

---

## Out of Scope

- RTL support (both EN and RU are LTR)
- More than 2 languages
- Backend translations (quiz content stays in original language)
